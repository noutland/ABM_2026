# ABM_2026
Shared repository for agent-based modeling class projects. Each team stores their NetLogo models, supporting materials, and documentation here.

## Repository Structure

Each team has their own folder at the root of this repository:

```
loafing/
human-ai-teaming/
neurodiv/
bobo/
```

Within your team folder, organize your materials something like this:

```
loafing/
  models/          # NetLogo model files (.nlogox)
  data/            # Input data or parameter files
  docs/            # Write-ups, reports, presentations
  results/         # Output data, plots, figures
  README.md        # Description of your project
```

You don't need to follow this exact structure, but each team folder should have a README that describes what your model does and how to run it.

## Getting Started

1. Install [GitHub Desktop](https://desktop.github.com/) if you don't already have it.
2. Make sure you have [NetLogo](https://ccl.northwestern.edu/netlogo/) installed.
3. Clone this repository using GitHub Desktop (see below).

## Adding Files with GitHub Desktop

If you're new to Git, GitHub Desktop is the easiest way to work with this repo. Here's the step-by-step:

### First time setup

1. Open GitHub Desktop
2. Go to **File > Clone Repository**
3. Select the **URL** tab and paste: `https://github.com/noutland/ABM_2026.git`
4. Choose where you want the folder to live on your computer and click **Clone**

### Adding or updating files

1. Before you do anything, click **Fetch origin** (top bar) to make sure you have the latest version. If there are changes, click **Pull origin**.
2. Open the repo folder on your computer (click **Repository > Show in Finder** on Mac, or **Show in Explorer** on Windows)
3. Navigate to your team folder (e.g., `loafing/`) and add or edit your files there — drag in your `.nlogox` files, data files, write-ups, etc.
4. Go back to GitHub Desktop. You should see your changes listed on the left side.
5. At the bottom left, write a short summary of what you changed (e.g., "added initial model draft" or "updated parameters for experiment 2")
6. Click **Commit to main**
7. Click **Push origin** to send your changes to GitHub

### Tips

- Always **Fetch/Pull** before you start working so you have everyone's latest changes
- Work within your own team folder to avoid stepping on other teams' files
- Write commit messages that are descriptive enough for your teammates to follow along
- If GitHub Desktop shows a merge conflict, don't panic — ask the instructor for help

## What to Include

At minimum, your team folder should contain:
- Your NetLogo model file(s)
- A README describing the model, what it simulates, key parameters, and how to run it
- Any data or supporting files needed to run or understand the model
